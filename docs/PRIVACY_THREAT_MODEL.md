# LocalFace Studio 隐私与威胁模型

状态：阶段 0 基线  
日期：2026-07-22  
适用范围：第一版单张照片、本地 Web 应用、原生 ONNX 后端，以及未来可选的本机 ComfyUI 适配器。

## 1. 安全目标

1. 未经用户明确授权确认，不处理人脸图片。
2. 默认情况下，人脸图片、身份表示和生成结果不离开本机。
3. 应用不持久化人脸身份向量。
4. 用户选择哪张目标脸，后端就只替换该目标脸。
5. 模型和依赖的来源可核验，篡改资产不得加载。
6. 本地服务不能因为监听范围、CORS 或主机头配置而被其他设备或恶意网页滥用。
7. 输出包含 AI 编辑声明，并默认提供可见水印。

## 2. 受保护资产

- 来源图片和目标图片。
- 任务中间图、人脸裁剪和输出结果。
- 人脸关键点、检测框和身份向量。
- 本机目录、用户名和设备信息。
- 模型文件及其哈希清单。
- 用户对目标人物的选择。
- 应用日志和错误报告。

## 3. 信任边界

### 边界 A：浏览器到本地 FastAPI 服务

浏览器输入不可信。即使服务只在本机运行，恶意网页也可能尝试访问本机端口，因此不能把“localhost”视为自动安全。

### 边界 B：应用到图片解码库

上传文件的扩展名、MIME 类型、元数据、尺寸和实际编码均不可信。

### 边界 C：应用到模型运行时

模型下载源、模型文件和运行时均属于供应链边界。未经哈希核验的模型不可信。

### 边界 D：原生后端到可选 ComfyUI

ComfyUI 是独立进程和独立许可证组件。其自定义节点、工作流和输出必须视为外部系统输入。

### 边界 E：项目目录到 GitHub

任何准备提交的文件都可能意外包含模型、图片、日志、绝对路径或密钥。

## 4. 数据流与保留

1. 浏览器上传图片到本地 API。
2. API 使用服务器生成的任务 ID 创建独立临时目录。
3. 校验通过后解码图片并执行检测；身份向量仅保存在当前进程内存。
4. 用户选择目标检测 ID 后继续换脸、融合和可选增强。
5. 导出副本写入 AI 编辑元数据，并按设置添加可见水印。
6. 用户下载结果。
7. 任务取消、失败、会话结束或超过保留期后删除任务文件。
8. 应用启动时扫描并清理上次异常退出遗留的过期任务目录。

第一版不得建立用户图库、身份库或历史任务数据库。

## 5. 威胁、缓解和剩余风险

| ID | 威胁 | 主要影响 | 必须实施的缓解 | 剩余风险 |
|---|---|---|---|---|
| T-01 | 用户上传未经授权的人脸 | 肖像权、隐私和滥用 | 强制授权确认、用途说明、默认水印、AI 元数据、禁止敏感演示素材 | 本地软件无法验证用户陈述真实性；公开部署前需要更强治理 |
| T-02 | 服务意外暴露到局域网或公网 | 他人访问图片和 GPU | 默认绑定 `127.0.0.1`；禁止 `0.0.0.0`；启动时显示地址；远程模式需单独设计 | 本机恶意软件仍可能访问本机服务 |
| T-03 | 恶意网页调用本地 API | 未经用户操作创建任务 | 精确 Origin/Host 白名单、禁止通配 CORS、状态变更请求使用会话令牌、拒绝不可信主机 | 浏览器或依赖漏洞仍可能绕过部分保护 |
| T-04 | 伪装图片、解压炸弹或超大图片 | 内存耗尽、解析器漏洞 | 检查魔数、限制字节和像素、限制帧数、设置解码警告为错误、尽早拒绝异常输入 | 第三方解码库仍可能出现未知漏洞 |
| T-05 | 文件名路径穿越或覆盖 | 读取/写入任意文件 | 永不使用用户文件名拼接路径；服务器生成文件名；所有路径解析后验证位于任务目录 | 实现错误仍需单元和集成测试覆盖 |
| T-06 | 临时文件或崩溃遗留 | 人脸数据残留 | 独立任务目录、过期清理、启动清理、失败路径 `finally` 清理、默认短保留期 | SSD 和文件系统可能保留不可由应用保证擦除的数据块 |
| T-07 | 身份向量被记录或复用 | 生物特征隐私风险 | 只在内存存在；不写磁盘、不序列化、不记录日志；任务结束释放引用 | Python/运行时不能保证物理内存立即清零 |
| T-08 | 模型或依赖被篡改 | 任意代码执行或结果操纵 | 官方来源、SHA-256、锁定依赖、拒绝未知哈希；默认优先 ONNX，不加载不可信 pickle | 官方上游或构建供应链仍可能受攻击 |
| T-09 | 任务选择错位 | 替换错误人物 | 任务内稳定检测 ID、选择后再次校验、UI 显示框、换脸前确认、回归测试 | 极端遮挡和相似人脸仍可能导致检测不稳定 |
| T-10 | GPU/内存耗尽 | 应用崩溃、系统不可用 | 单任务队列、尺寸限制、资源预算、取消、超时、异常后释放会话 | 驱动级故障可能需要重启进程 |
| T-11 | 日志、遥测或错误页面泄露 | 路径和人脸信息泄露 | 默认无遥测；结构化日志脱敏；生产界面隐藏堆栈；错误导出需用户确认 | 用户主动分享日志时仍需人工检查 |
| T-12 | 水印被关闭或移除 | 结果被误认为真实图片 | 默认开启、关闭时二次提示、元数据始终写入、界面保留处理记录 | 元数据和像素水印都可以被后续工具移除 |
| T-13 | ComfyUI 自定义节点执行任意逻辑 | 代码执行、联网和数据泄露 | 进程隔离、仅本机 API、工作流和节点允许清单、默认不开启、单独模型目录 | 第三方节点本质上是可执行代码，不能在未审计时信任 |
| T-14 | 敏感文件进入 GitHub | 永久公开泄露 | `.gitignore`、提交前扫描、模型和数据目录在仓库外或明确排除、首次推送人工复核 | 已推送历史不能只靠删除最新文件解决 |

## 6. 默认安全配置

- 网络：仅 `127.0.0.1`。
- 并发：第一版一次只运行一个推理任务。
- 遥测：关闭。
- 身份向量持久化：禁止。
- 历史记录：关闭。
- 可见水印：开启，可由用户关闭。
- AI 元数据：始终开启。
- ComfyUI：默认关闭。
- 商业模式：默认关闭，且受限模型后端硬性禁止在商业模式加载。

## 7. 隐私测试要求

- 搜索任务目录，确认过期任务已删除。
- 扫描日志，确认不存在图片字节、身份向量和绝对用户路径。
- 使用错误扩展名、超大尺寸、损坏文件和多帧文件测试输入限制。
- 从非允许 Origin 和 Host 调用 API，确认被拒绝。
- 终止进程后重启，确认遗留临时任务被清理。
- 对所有失败和取消路径验证清理逻辑。
- 在 Git 提交前执行敏感文件与大文件扫描。

## 8. 不作出的安全承诺

- 授权勾选不能证明照片主体真实同意。
- 本地运行不能抵御已经控制本机的恶意软件。
- 普通文件删除不等同于取证级安全擦除。
- 水印和元数据不能保证永远不被移除。
- 自动身份相似度不能判断法律意义上的人物身份。

## 9. 公开或商业部署前的额外门禁

- 完成独立法律与隐私审查。
- 替换所有非商业模型。
- 增加认证、授权、速率限制、审计、滥用举报和删除机制。
- 制定未成年人、敏感内容和非自愿亲密内容政策。
- 评估适用地区的合成媒体标识、肖像权和生物特征信息要求。
- 进行第三方安全测试和依赖漏洞扫描。
