# LocalFace Studio 项目路线图

状态：2026-07-22 已获产品负责人确认  
执行规则：每个阶段完成、测试并由琪琪验收后，才进入下一阶段。

## 总体原则

- 产品优先，研究能力作为可验证的扩展。
- 默认本地运行、严格零付费、人脸数据不离开设备。
- 先建立可测试的产品壳，再接入真实模型。
- 先交付单张照片，再扩展视频。
- 原生后端是默认路径；ComfyUI 是可选独立后端。
- 当前研究权重与未来商业产品严格隔离。
- 所有面向用户的失败必须可解释，不隐藏降级或伪造成功。
- 使用单一公开 GitHub 仓库展示项目演进，每次推送前执行敏感内容、大文件和二进制扫描。
- 原始旧报告不上传；公开仓库只保留已脱敏的 Markdown 复盘。

## 阶段 0：策划冻结与合规基线

目标：把已确认的口头策划转化为版本可追踪的工程约束。

交付物：

- 产品需求文档。
- 项目路线图。
- 混合后端架构决策。
- 许可证清单。
- 隐私与威胁模型。
- 第一版验收标准。

退出条件：文档之间不存在范围、许可证或安全策略冲突，并由琪琪确认。

## 阶段 1：工程骨架与独立环境

目标：创建无需模型即可启动、测试和维护的前后端工程。

工作项：

- 创建项目目录内 `.venv`。
- 锁定 Python 与前端依赖。
- 建立 FastAPI 后端、React/TypeScript/Vite 前端和共享契约。
- 建立配置、结构化日志、测试、格式化和静态检查。
- 配置 `.gitignore`，排除模型、图片、缓存、身份向量和结果。
- 初始化本地 Git，并连接公开 GitHub 仓库；首推送前完成全量内容扫描。
- 提供 Windows 启动与诊断脚本。

退出条件：新环境可重复安装，前后端健康检查和基础测试通过。

## 阶段 2：模拟推理产品闭环

目标：在不下载模型的情况下打通真实产品流程。

冻结范围：

- 上传来源图与目标图；支持 PNG、JPEG 和静态 WebP。
- 单图上限 25 MB、5000 万像素、最长边 12000 px；拒绝损坏、伪装格式、动画和多帧图片。
- 每次创建任务前强制确认人像授权与合法用途；保存最小化、带版本的授权回执。
- 使用随机本地会话、CSRF 令牌和 Host/Origin 校验；通过 `actor_id` 预留未来账号绑定能力。
- 使用 HTTP 创建、取消、下载和删除任务，使用 SSE 单向推送后端任务事件。
- 任务状态包括排队、处理、成功、失败、取消、超时、过期和删除。
- 可视化节点包括文件校验、任务准备、模拟处理、输出检查和导出结果。
- 模拟后端复制目标图并添加醒目的“SIMULATION—非真实换脸结果”标识，不伪装真实换脸。
- 默认导出 PNG，可选 JPEG；JPEG 质量可在 5–100 间调整、默认 95；可见水印默认开启、允许关闭，AI/模拟处理元数据始终写入。
- 提供滑杆对比、并排查看、同步缩放和平移，以及明确的剩余保留时间。
- 默认保留 30 分钟，可显式选择 1、3、6、12 或 24 小时；24 小时为硬上限。
- 使用 SQLite 保存最小任务记录，图片存入随机隔离目录；不保存人脸特征、身份向量或图片二进制。
- 同时只处理一个任务；每个本地会话最多保留三个未完成任务。
- 故障注入仅供自动化测试和默认关闭的开发模式使用，不暴露在普通产品界面。
- 通过统一任务和后端契约预留真实检测、换脸引擎、ComfyUI、视频、账号和服务器数据库扩展。

退出条件：

- 模拟成功、失败、取消、超时、过期清理、越权访问和恶意上传均有自动化测试。
- 前端任务与节点状态完全来自后端事件，刷新后可恢复或明确说明任务已失效。
- 输出尺寸、格式、水印和元数据符合产品需求，且模拟结果不会被误认为真实换脸。
- 失败、取消和超时不留下可下载的伪成功结果，所有终止路径均进入清理流程。

## 阶段 3：人脸检测与目标选择

目标：稳定处理来源人脸与目标图多脸选择。

工作项：内容级图片校验、人脸检测、检测框、稳定编号、来源歧义处理、目标选择、检测 ID 校验。

退出条件：单脸、多脸、无脸、小脸、遮挡和旋转测试均产生明确且正确的行为。

## 阶段 4：原生精准换脸 MVP

目标：在目标设备上完成第一条真实 GPU 换脸路径。

工作项：模型清单与哈希、许可证门禁、ONNX 执行器、对齐、换脸、融合、错误处理、GPU/CPU 策略、资源释放和性能基线。

退出条件：所选人脸正确替换，未选区域保持，任务失败不产生伪结果，目标设备连续运行测试通过。

## 阶段 5：质量优化

目标：在不牺牲身份一致性的前提下改善边缘、颜色和小脸质量。

工作项：颜色匹配、遮罩羽化、可选增强、增强前后对比、参数预设和回归样例。

退出条件：每项优化均有独立消融证据；无证据的增强不得默认开启。

## 阶段 6：质量评测

目标：建立与研究问题相匹配的评价框架。

工作项：检测成功率、对齐后身份相似度、关键点保持、换脸区域外变化、分辨率与格式检查、盲评表。

退出条件：不再使用全图 LPIPS 冒充身份指标；自动分数的适用范围和限制均被记录。

## 阶段 7：隐私、水印与安全验收

目标：验证数据生命周期和输出标记真实有效。

工作项：本机绑定、文件限制、任务隔离、过期清理、敏感日志测试、AI 元数据、默认可见水印和关闭选项。

退出条件：威胁模型中的高风险项均已缓解或由产品负责人书面接受。

## 阶段 8：ComfyUI 可选适配器

目标：通过进程外 API 接入 ComfyUI，不污染默认后端。

工作项：健康检查、工作流提交、状态映射、结果获取、超时和断线处理、许可证边界验证。

退出条件：没有 ComfyUI 时默认产品功能完整；适配器失败不会导致主服务不可用。

## 阶段 9：Windows 一键运行与文档

目标：让新用户可以按文档独立安装、运行和排错。

工作项：安装、模型检查、一键启动、诊断、用户手册、开发文档、许可证归档、测试素材清单和实测报告。

退出条件：在干净环境按文档完成复现，不依赖开发者口头补充。

## 阶段 10：GitHub 发布与视频接口准备

目标：完成公开仓库的版本发布准备，并冻结视频扩展契约。

工作项：公开仓库发布说明、Issue 模板、拉取与回滚说明、社区文档，以及帧源、跟踪器、时间一致性、音视频复用和中断恢复接口。

退出条件：仓库扫描不含模型或人脸数据；图片版本完成标准全部满足；视频功能仍保持在明确的下一版本范围。

## 变更控制

以下变更必须先更新文档并取得琪琪确认：

- 从本地转为联网或云端处理。
- 从非商业演示转为商业使用。
- 改变默认模型、下载源或模型许可证。
- 开放公网、局域网或远程访问。
- 持久化身份向量或用户图片。
- 将视频、实时摄像头或多人批量替换提前纳入第一版。
