## 遇到的问题及处理

### PowerShell 默认编码显示中文乱码

文件本身为 UTF-8，问题来自旧式 PowerShell 默认读取编码。验证命令改为显式指定 UTF-8，文件内容无需重写。

### `rg` 在当前沙箱中被拒绝执行

改用 PowerShell `Select-String` 和本地路径检查完成一致性验证，不影响文档结果。

### 本机缺少 GitHub CLI

已在项目目录内安装并校验便携版 GitHub CLI，但设备授权请求两次连接 GitHub 超时。用户已人工建立公开空仓库 `Wenqi77Zhang/localface-studio`；发布改用已授权的 GitHub 连接通道，不使用个人访问令牌，也不降低仓库隐私与提交检查标准。

### 本地提交与远端首次发布的提交标识不同

GitHub 连接通道不能直接推送本地 Git 对象，因此远端首次发布会生成新的提交标识。文件内容仍以本地审查通过的版本为准；网络恢复后再通过普通 Git 拉取并对齐历史，后续开发恢复标准分支与推送流程。
