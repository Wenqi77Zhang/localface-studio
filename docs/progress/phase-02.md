# 阶段 2 进展日志：模拟推理产品闭环

状态：进行中（策划已冻结）  
开始日期：2026-07-23  

## 阶段目标

在不下载或运行人脸模型的前提下，完成可实际操作的单图产品闭环，验证上传、授权、任务队列、实时事件、节点可视化、结果比较、下载和清理等工业化边界。

## 已冻结决策

| 主题 | 决策 |
|---|---|
| 模拟结果 | 复制目标图并添加醒目的“SIMULATION—非真实换脸结果”标识 |
| 上传格式 | PNG、JPEG、静态 WebP |
| 上传限制 | 单图 25 MB、5000 万像素、最长边 12000 px |
| 本地会话 | 随机会话、CSRF 防护、Host/Origin 校验 |
| 账号扩展 | 使用 `actor_id` 预留账号绑定和最小化授权审计 |
| 实时事件 | SSE 推送；HTTP 负责创建、取消、下载和删除 |
| 任务状态 | 排队、处理、成功、失败、取消、超时、过期、删除 |
| 工作流节点 | 文件校验、任务准备、模拟处理、输出检查、导出结果 |
| 输出格式 | PNG 默认，可选 JPEG |
| 输出标记 | 可见水印默认开启、允许关闭；AI/模拟元数据始终写入 |
| 结果比较 | 滑杆、并排模式、同步缩放和平移 |
| 默认保留 | 30 分钟 |
| 可选保留 | 24 小时或 7 天；7 天为硬上限 |
| 任务记录 | SQLite 最小化记录；图片使用随机隔离目录 |
| 并发策略 | 单任务执行；每个会话最多三个未完成任务 |
| 故障测试 | 仅自动化测试和默认关闭的开发模式可用 |

## 隐私与安全约束

- 每个任务必须重新确认图片授权和合法用途。
- 授权回执只保存会话或账号标识、确认时间、声明版本和任务编号。
- 本地版不记录 IP，不在日志中记录图片内容、原始文件名、绝对路径、令牌或人脸特征。
- 原始图、目标图和结果图仅存在于当前随机任务目录。
- 文件扩展名、媒体类型和解码后的真实格式必须一致。
- 拒绝损坏图片、动画或多帧图片、超限图片、压缩炸弹和路径穿越。
- 失败、取消和超时立即清理任务图片；启动时清理过期遗留任务。
- 模拟输出必须同时包含可见模拟标识和 `simulation=true` 元数据。

## 阶段退出门禁

- [ ] 模拟成功、失败、取消和超时路径均通过自动化测试。
- [ ] 过期清理、越权访问、路径穿越和恶意图片均通过安全测试。
- [ ] 前端任务与节点状态和后端 SSE 事件一致。
- [ ] 刷新页面后可恢复有效任务，或明确说明任务已经失效。
- [ ] PNG/JPEG 输出尺寸、可见标识、AI 元数据和水印策略通过验证。
- [ ] 所有终止路径不产生可下载的伪成功结果，并按策略完成清理。
- [ ] 产品负责人在目标设备上完成阶段 2 人工验收。

## 变更控制

阶段 2 的冻结范围可以通过后续架构决策调整，但涉及联网、账号身份收集、IP 记录、持久化人脸数据、放宽保存期限或公开部署的变更，必须先更新隐私威胁模型并由产品负责人明确确认。
