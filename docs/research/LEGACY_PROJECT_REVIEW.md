# 旧项目复盘：从提示词消融实验到精准换脸产品

状态：已脱敏公开版本  
复盘日期：2026-07-22  

## 1. 文档目的

本文件记录 LocalFace Studio 的历史起点、可继承经验和需要修正的方法学问题。旧报告原件包含个人信息、人脸素材和来源许可尚未核验的图片，因此不进入公开 GitHub 仓库。

原件只保存在私有位置，其完整性校验信息也不进入公开仓库。

## 2. 旧项目概述

旧项目使用 SDXL、双 ControlNet、IPAdapter FaceID、Florence-2、FaceDetailer 和 PyIQA，尝试比较三种提示词策略：简短人工提示词、详细人工提示词和视觉语言模型自动提示词。

旧项目希望回答：提示词复杂度是否会在整体美学质量与面部身份保持之间形成权衡。

## 3. 值得继承的内容

- 已形成结构控制、身份注入、生成和评价的模块化思维。
- 意识到面部身份保持和整体画面质量需要分开评价。
- 观察到光照、场景描述和身份条件可能发生冲突。
- 使用统一参数对比不同提示策略，具备消融实验意识。
- ComfyUI 节点图能够清晰展示数据流和实验逻辑。

## 4. 主要方法学问题

### 4.1 身份指标不成立

旧实验把完整生成图与近景来源人脸直接计算全图 LPIPS。三组背景、服装和光照并不相同，因此背景差异不是常量；LPIPS 也不是人物身份识别指标。相应结果不能证明某一组身份保持最好。

### 4.2 样本量不足

实验只展示单一来源身份、单一场景和每组单一结果，没有多随机种子、方差、置信区间或显著性检验，不能支持普遍性或因果性结论。

### 4.3 自变量混杂

三组提示词不仅长度不同，还同时改变了光照、服装、背景语义和风格。结果不能单独归因于“提示词复杂度”。

### 4.4 后处理未单独消融

FaceDetailer 会再次生成面部区域，可能改善清晰度，也可能改变身份。旧报告没有分别评价增强前后结果。

### 4.5 指标解释过度

MUSIQ、BRISQUE、LAION aesthetic predictor 和 CLIP-IQA 都有明确适用范围，不能直接等同于艺术性、真实性或客观审美真值。数值差异也没有误差范围支持。

### 4.6 复现信息不足

报告缺少完整 workflow JSON、节点与依赖版本、模型来源与哈希、精确种子、环境锁定文件和可执行复现步骤。截图不能替代机器可读工作流。

### 4.7 安全与许可缺失

旧报告没有系统记录图片授权、人脸数据生命周期、模型许可证、深度伪造滥用、水印或输出声明。

## 5. 新项目的对应改进

| 旧问题 | 新项目改进 |
|---|---|
| 全图 LPIPS 冒充身份指标 | 检测并对齐人脸后评价身份相似度，并明确指标限制 |
| 单一样本 | 建立多姿势、多光照、多人物和多脸测试矩阵 |
| 提示词混杂 | 第一版先取消全图生成，聚焦传统精准换脸 |
| FaceDetailer 混杂 | 基础结果与增强结果分开保存、分开评价 |
| ComfyUI 强耦合 | 原生后端默认运行，ComfyUI 通过独立 API 适配 |
| 不可复现 | 锁定依赖，登记模型来源、许可证与 SHA-256 |
| 无隐私治理 | 本机绑定、临时文件清理、身份向量不落盘、水印与元数据 |
| 结论过强 | 使用失败案例、误差范围和人工盲评限制结论强度 |
