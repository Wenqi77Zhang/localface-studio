# LocalFace Studio 许可证与资产清单

状态：阶段 0 基线  
核验日期：2026-07-22  

> 本清单用于工程风险控制，不构成法律意见。每次锁定或升级依赖时都必须重新生成依赖清单并复核许可证。

## 1. 判定标签

- **绿色**：候选许可证通常允许商业使用和再分发，但仍须保留版权与许可证文本。
- **黄色**：可以按隔离方式使用，或需要额外确认分发、衍生作品和模型条款。
- **红色**：当前权重禁止作为商业产品默认资产或随仓库/安装包分发。
- **待核验**：在取得明确官方许可证前不得进入默认依赖或模型下载流程。

## 2. 本项目代码许可证

拟采用：MIT。

阶段 1 初始化仓库时再添加正式 `LICENSE` 文件。在此之前必须确认：

- 没有复制 GPL-3.0 代码到本项目。
- 没有把许可证不兼容的代码片段合并到核心代码。
- 第三方版权声明能够随发布物保留。

### InsightFace 权重的关键限制

InsightFace 官方明确区分：源代码采用 MIT；官方提供或自动下载的预训练模型仅限非商业研究。官方还要求 `inswapper` 系列的商业使用方联系其授权渠道。因此，项目不得因为 Python 包是 MIT 就宣称模型可商用。

官方依据：

- https://github.com/deepinsight/insightface
- https://github.com/deepinsight/insightface/blob/master/python-package/README.md#license

### SimSwap 的关键限制

SimSwap 官方项目采用 CC BY-NC 4.0，并明确限定学术和非商业用途。它不能作为满足未来商业目标的替代方案。

官方依据：https://github.com/neuralchen/SimSwap

## 5. 模型获取与加载门禁

阶段 4 的实现必须满足：

1. GitHub 仓库和项目安装包均不包含受限模型。
2. 下载器只接受许可证清单中登记的官方来源。
3. 每个模型记录文件名、来源 URL、发布日期或版本、许可证文本版本、SHA-256 和期望大小。
4. 哈希不匹配时拒绝加载，不能静默继续。
5. 首次启用红色模型前显示非商业限制并取得明确确认。
6. 后端响应和导出元数据记录模型标识，但不泄露本机绝对路径。
7. 商业模式开关不得加载红色模型；这一约束必须由后端执行，不能只依赖前端提示。

## 6. GitHub 与分发规则

禁止提交：

- 模型权重及其压缩包。
- 自动下载的模型缓存。
- 用户图片、测试人物原图和生成结果。
- 人脸嵌入、检测缓存和任务临时目录。
- 包含个人路径、访问令牌或隐私数据的日志。

允许提交：

- 模型清单模板。
- 官方下载说明和许可证链接。
- 不含权重的适配器代码。
- 经过许可登记的极小测试夹具；优先使用程序生成的非人物图进行结构测试。

## 7. 商业化前的重新审查清单

- 获得换脸、检测、对齐、识别和增强权重的商业授权证明。
- 核验训练数据权利不会把额外限制传递到模型。
- 替换所有红色资产并重新跑完整回归测试。
- 确认测试和宣传图片具备商业使用及人物肖像授权。
- 审查 ComfyUI 的部署和分发方式。
- 进行目标司法辖区的隐私、肖像权、消费者保护和合成媒体标识审查。
